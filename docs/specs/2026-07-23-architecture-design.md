# Architecture Design: B题模型架构规格

> 决策日期: 2026-07-23 | 状态: 已敲定

---

## 1. 背景约束

- 4380行训练数据（2025全年），36行测试数据（2026年1-3月）
- 全部模型验证在2025数据上做TimeSeriesSplit，2026仅做最终预测演示
- 物理约束不出现在架构图中作为独立组件——溶解在数据层→特征层→模型层→元特征层→输出层
- 融合范式：RF元学习器在特征空间做条件推理（非输出加权平均）

## 3. 物理约束完整溶解路径（7层）

| 层 | 物理形式 | 数学表达 | 不可绕过性 |
|:---:|------|------|:---:|
| 1.数据 | 合理性筛选 | FILT ≤ R/W (0违规率), NTU ≥ 0, |ΔNTU|分布 | 硬 |
| 2.特征 | 物理定义 | η_coag = (R/W-FILT)/R/W (质量守恒); Π_load = NTU·Flow (通量) | 软 |
| 3.模型Loss | 处罚违反 | L+=0.1·Σ|Δŷ|(平滑) + 0.5·ΣReLU(ŷ-X_raw)(上界) + 0.5·ΣReLU(-ŷ)(非负) | 硬 |
| 4.模型架构 | 结构保证 | TCN因果卷积(时间不泄露); GRU遗忘门b_f=+1.0(水质惯性) | 硬 |
| 5.元特征 | 违规告知 | d_multi, d_uni, d_neg, safety → RF能看到谁在违反物理 | 软 |
| 6.输出 | 硬裁剪 | clip(ŷ_final, 0, X_raw(t-τ₁)) | 硬 |
| 7.训练数据 | 异常降权 | y_true > X_raw(t-τ₁) → 权重×0.1 (标注噪声或极端事件) | 软 |

**设计哲学**：物理不出现在架构图中作为独立组件。它不输出预测值，只做两件事——约束（如果模型试图违反物理，惩罚或裁剪它）和告知（告诉RF此刻谁在越界）。

## 4. 各小问 Loss 函数汇总

### 4.1 Q1 Loss

```python
L_Q1 = Huber(delta=1.0)(ŷ, y)  # 时序列Huber，对异常值鲁棒
```

### 4.2 Q2 Loss

```python
L_Q2 = Huber(delta=1.0)(ŷ_filt, y_filt)
     + 0.1 * mean(|ŷ_t - ŷ_{t-1}|)                     # 平滑性: 浊度不跳变
     + 0.5 * mean(ReLU(ŷ_filt - X_raw(t-τ₁)))          # 物理上界: 过滤不增浊
```

### 4.3 Q3 Loss（统一目标）

```python
L_Q3 = sum_{h=1}^{6} Huber(delta=1.0)(ŷ(t+h), y(t+h))  # 多步预测
     + 1.0 * sum_{h} ReLU(ŷ(t+h) - ŷ_phys_bound)         # 物理上界(强)
     + 0.1 * sum_{h=2}^{6} |ŷ(t+h) - ŷ(t+h-1)|            # 时序平滑
     + 0.01 * |attention - oracle_trust|                   # 一致性(最弱)
```

**λ调参优先级**：λ₃(物理上界=1.0) > λ₁(平滑=0.1) > λ₂(一致性=0.01)

### 4.4 Q4 评分公式

```
S_risk(t) = w₁·f₁'(t) + w₂·f₂(t) + w₃·f₃'(t)

其中:
  f₁ = max(0, (NTU-1.0)/1.0) / P99        ← 超标幅度(归一化)
  f₂ = 1 - exp(-γ·D_run), γ = ln2/T_half  ← 持续时长(指数饱和, T_half=6h)
  f₃ = ReLU(NTU(t) - NTU(t-3)) / (3Δt)     ← 恶化趋势(仅上升罚分)
  w_i = (1-e_i) / Σ(1-e_j), e_i为信息熵    ← 熵权法客观赋权
```

## 5. 消融实验矩阵（Q3专属）

```
┌────────────┬──────────┬───────────┬──────────┬──────────────────┐
│  单变量插槽  │ TCN→GRU  │ RF元学习器 │  输出     │ 回答什么问题       │
├────────────┼──────────┼───────────┼──────────┼──────────────────┤
│  空         │    ✅    │    ✅     │ ŷ_A+RF   │ B源贡献度          │
│  N-BEATS   │    ✅    │    ✅     │ ŷ_full   │ 默认配置           │
│  TimesFM   │    ✅    │    ✅     │ ŷ_full+  │ 大模型增益          │
│  仅N-BEATS │    ❌    │    ❌     │ ŷ_uni    │ 纯自回归上限        │
│  仅TimesFM │    ❌    │    ❌     │ ŷ_tfm    │ 纯零样本上限        │
│  仅RF      │    ❌    │    ✅     │ ŷ_meta   │ 元特征本身的信息量   │
└────────────┴──────────┴───────────┴──────────┴──────────────────┘

附加消融（Q2）:
  消融1: 移除物理Loss(仅MSE) → 纯TCN
  消融2: 传递熵→互信息(无方向) 
  消融3: TCN→简单AR(6)
  消融4: 移除时滞注意力
  
附加消融（Q1特征层）:
  消融L5→L4→L3→L2: 逐层移除，量化每层贡献
```

## 6. 关键接口定义

### 6.1 特征工程输出

```
step0_preprocess.py → X_train, y_train, X_test (2026)
  X: [n_samples, ~60 features]  L1-L5五级金字塔
  y: [n_samples] 对应目标变量
  保存: data/processed/X_train.npy, y_train.npy
```

### 6.2 Q1→Q2 传递

```
step1.0_feature_selection.py → I_j^robust (特征重要性向量)
  → step2.0/2.1 使用 I_j > 0.05 的特征子集构建输入
```

### 6.3 Q2→Q3 传递

```
step2.0_time_delay_estimation.py → τ* = {τ₁(R/W NTU), τ₂(pH), τ₃(ALUM固定6h), τ₄(FLOW)}
  → step3.0 使用 τ* 进行输入时间对齐
```

### 6.4 Q3→Q4 传递

```
step3.2_meta_feature_matrix.py → ŷ_final(t+1:t+H) 预测值
  → step4.0 使用预测值计算 S_risk(t)
```

## 7. 构建与验证

### 7.1 验证策略

- **主验证**：2025年数据 TimeSeriesSplit(n_splits=5)
- **最终测试**：2026年36行数据（不参与任何训练/调参，仅一次预测演示）
- **指标**：RMSE(报告), R²(报告), Huber Loss(训练), MAPE(补充)
- **论文声明**："由于2026年测试集仅有36条记录，无法进行统计检验。所有模型比较和消融实验均在2025年数据的5折时序交叉验证上完成。"

### 7.2 成功标准

| 小问 | 最低标准 | 目标标准 |
|:---:|------|:---:|
| Q1 | R² > 0.6 on 2025 CV | R² > 0.75 |
| Q2 | RMSE < 0.15 NTU on FILT | RMSE < 0.10 |
| Q3 | RMSE < 0.20 NTU @6h ahead | RMSE < 0.15 |
| Q4 | Kappa > 0.7 (FCE vs Jenks) | Kappa > 0.85 |
