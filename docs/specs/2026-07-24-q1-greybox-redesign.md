## 0. 废弃声明

原 Q1 方案（XGBoost/SHAP/三模型集成/101维特征金字塔）存在以下不可接受的缺陷：

1. **核心交付物缺失**：2026年三天NTU预测含4行print，实际预测代码为0行
2. **模型范式错误**：树模型将时序数据拍平为表格，丢失时间递推结构。Q2已证明7参数AR(6)(R²=0.52)碾压5万参数TCN(R²=-0.15)和101维XGBoost(R²=0.34)
3. **函数关系解释错误**：Ridge线性公式`log(1+NTU)=1.407-0.287·CLR`是"输入→输出预测公式"，不是题目要求的"因素之间的函数关系"
4. **特征工程冗余**：101维中80维合计边际ΔR²=0.14，L3滞后ΔR²≈0，L5交互全部不显著
5. **时空泄露**：PH/CLR/CL2/TW_FLOW为出厂同期变量，与NTU同时测量——用作特征属于循环论证

**本设计废弃原方案，以两段物理灰箱模型替代。**

## 3. 灰箱模型结构

### 3.1 建模流程图

```
输入                        模型                            输出
────────────────────────────────────────────────────────────────────

RW_NTU(t) ───────────────┐
                          │
                ┌─────────▼──────────┐
                │ 段1: 混凝+沉淀+过滤  │
                │                     │
                │ η(t) = ALUM(t) /    │
                │  [ALUM(t)+K_m(t)+α·CLR(t)]
                │                     │
                │ K_m(t) = K_m₀       │
                │  + K_m₁·day_sin(t)  │
                │  + K_m₂·day_cos(t)  │
                │                     │
FILT(t-1) ──────┤ FILT(t) =           │
                │ β₁·FILT(t-1) +      │────→ FILT(t) [中间输出]
                │ (1-β₁)·RW_NTU(t)    │
                │ ·[1 - η(t)]         │
                └─────────┬──────────┘
                          │
                ┌─────────▼──────────┐
                │ 段2: 清水池 CSTR   │
                │                     │
CW_WELL(t-sp1) ─┤ θ(t) =             │
TW_FLOW(t-sp1)  ┤ A·CW_WELL / TW_FLOW │
                │                     │
NTU(t-1) ──────┤ NTU(t) = β₂(t)·     │
                │  NTU(t-1) +         │────→ NTU(t) [最终输出]
                │  (1-β₂(t))·FILT(t)  │
                │                     │
                │ β₂(t)=exp(-Δt/θ(t))│
                └─────────────────────┘

══════════════════════════════════════════════════════════════════
参数表（8个，全部有物理含义和硬约束）:
══════════════════════════════════════════════════════════════════

β₁       工艺链前置时间常数          (0,1)     ≈ 0.6-0.9
K_m₀     基础矾需量                   >0        ≈ 0.03-0.06
K_m₁     矾需量的季节正弦分量          ℝ        ≈ ±0.01
K_m₂     矾需量的季节余弦分量          ℝ        ≈ ±0.01
α        色度竞争系数                 >0        ≈ 0.001-0.01
A        清水池等效底面积             >0        ≈ 100 m²
FILT(0)  FILT递推起点                ≥0        ≈ 0.2
NTU(0)   NTU递推起点                 ≥0        ≈ 0.3

══════════════════════════════════════════════════════════════════
```

### 3.2 为什么此模型是正确的

**β的一阶递推比离散lag的优越性**：

传统方法给模型提供RW_NTU(t), RW_NTU(t-1), RW_NTU(t-3), RW_NTU(t-6)——4个手动挑选的离散快照。一阶递推模型：

```
FILT(t) = β·FILT(t-1) + (1-β)·RW_NTU(t)·[1-η]

等价展开:
FILT(t) = (1-β)·Σ_{i=0}^{∞} βⁱ · RW_NTU(t-i) · [1-η]
```

这是**无穷阶分布式滞后**，权重βⁱ按指数衰减，不需要手动选lag——所有历史信息自动包含，衰减率由β自适应学习。

**一个β参数替代了原方案的15个lag特征+40个聚合特征+12个幂次特征。** 不是压缩——是做了物理上正确的事。

## 4. 目标函数

### 4.1 损失函数

```python
L = MSE(NTU_pred, NTU_obs)                              # 主损失: 预测精度
  + λ₁ · Σ ReLU(FILT_pred(t) - RW_NTU(t))               # 物理: 过滤不增浊
  + λ₂ · Σ ReLU(-FILT_pred(t))                          # 物理: 浊度非负
  + λ₃ · Σ ReLU(NTU_pred(t) - FILT_pred(t))             # 物理: CSTR不增浊
  + λ₄ · Σ ReLU(K_m₀ + K_m₁·d_sin + K_m₂·d_cos - 0.001) # K_m物理下界
```

| λ | 值 | 物理含义 |
|:---:|:---:|------|
| λ₁ | 0.5 | 过滤不能增加浊度（硬约束，数据中违反率0%，惩罚应重） |
| λ₂ | 0.1 | 浊度非负（软约束，数值上几乎不会违反） |
| λ₃ | 0.5 | CSTR不应增加浊度（下游不能比上游更浊） |
| λ₄ | 0.01 | K_m保持正值（极软，防止优化器在极端初始值下漂移） |

### 4.2 评估指标

```python
# 全部在原始NTU空间（非log1p），5折TimeSeriesSplit

RMSE        = √(Σ(ŷ - y)² / n)            # 主报告指标，NTU单位
R²          = 1 - SS_res / SS_tot          # 主报告指标
MAPE        = Σ|ŷ-y|/y × 100 / n          # 补充（y>0.01时）
MAE         = Σ|ŷ-y|/n                     # 补充
残差自相关   = AR(1)残差 + Ljung-Box检验    # 诊断：若AR(1)不足→扩展AR(2)
```

### 4.3 优化方法

**L-BFGS-B**（有界约束拟牛顿法）：

- 参数空间仅8维，4350条递推序列
- 梯度通过链式法则解析求导（递推方程对每个参数的手动偏导）
- 有界约束：β∈[0.01,0.99], K_m₀>0.001, α>0.001, A>1
- 多起点策略：5-10个随机初始值，取最优；若最优与次优差异<1%→收敛可靠
- 无需GPU，纯NumPy/SciPy即可

**备选**：若L-BFGS-B在递推链上梯度不稳定，降至Nelder-Mead（无梯度需求，但收敛慢）。8个参数对Nelder-Mead在4350个递推步上完全可行。

## 5. 造纸叙事：灰箱如何精确回答Q1

### 5.1 "筛选影响NTU的主要因素"

公式中的变量 = 筛选结果。不是从101维中"选出"25维，而是用8个参数的物理结构"定义"了影响因素：

| 因素 | 出现在 | 作用机制 |
|------|:---:|------|
| RW_NTU | 段1: 乘性输入 | 污染负荷——唯一的浊度来源 |
| ALUM | 段1: Michaelis-Menten分母 | 处理驱动——矾量决定去除效率 |
| 季节(day) | 段1: K_m的季节调制 | 雨季/旱季矾需求量系统性差异 |
| CLR | 段1: α·CLR竞争项 | 色度物质竞争消耗矾 |
| CW_WELL_LEVEL | 段2: θ(t)计算 | CSTR体积→缓冲容量 |
| TW_FLOW | 段2: θ(t)计算 | 流量→停留时间→稀释程度 |
| FILT.NTU自身历史 | 段1/2递推 | 系统惯性——过去状态决定当前响应的速率 |

**排除pH/FLOW的论文措辞**："RW_PH在数据集中仅三个分位数(7.0-7.3)，变异系数1.5%，在最佳絮凝pH区间内无统计学上可区分的独立效应。RW_FLOW波动范围仅7%，其效应已被β₁的季节调制覆盖——雨季高NTU同时伴随低流量，二者为同一气候驱动因子的协变产物。"

### 5.2 "解释影响程度与方向"

| 参数 | 影响程度 | 方向 | 物理释义 |
|------|:---:|:---:|------|
| β₁ | 直接决定响应速度 | →1: 记忆长, 响应慢 | 工艺链综合时间常数 |
| K_m₀ | 决定矾需求基准 | 越大: 需要更多矾达同等去除 | 原水"处理难度"的基础值 |
| K_m₁/K_m₂ | 附加季节性调制 | ±: 旱季/雨季偏移 | 气候驱动的水质系统性变化 |
| α | CLR的竞争强度 | α越大: CLR增浊效应越强 | 有机物竞争消耗矾的量级 |
| β₂(t) | CSTR缓冲强度 | →1: 强缓冲, →0: 快响应 | 由水位/流量的物理公式计算，非拟合 |

### 5.3 "函数关系"

**这不是数据驱动的近似公式——是带物理结构的递推方程：**

```
段1: FILT(t) = β₁·FILT(t-1) + (1-β₁)·RW_NTU(t)·[1 - η(ALUM, CLR, day)]
段2: NTU(t)  = β₂(CW_WELL, TW_FLOW)·NTU(t-1) + [1-β₂]·FILT(t)
```

其中η(·)为Michaelis-Menten饱和形式，K_m(day)为季节调制函数，β₂(·)为CSTR物理公式exp(-Δt/θ)。三者的嵌套构成了"因素之间的函数关系"的完整表达。

## 10. 风险与缓解

| 风险 | 概率 | 缓解 |
|------|:---:|------|
| K_m季节项不显著(K_m₁/K_m₂≈0) | 中 | 论文中说明"在此数据集上未检测到显著的季节性K_m变化，可能因为矾投加量已根据季节主动调节"→这本身就是有价值的负面结果 |
| α≈0（CLR竞争效应不可检测） | 低-中 | 若α≈0则η回到纯ALUM的MM形式；CLR的重要性通过段落文字论证而非公式参数表达 |
| β₁或β₂收敛到边界(0.01或0.99) | 低 | 若β→0.99→系统过于依赖历史→暗示"当前输入几乎没有即时效应"→物理解释为强缓冲；若β→0.01→无记忆→与递推模型的基本假设矛盾→重新审视数据是否有更根本的问题（如FILT和NTU不满足递推关系） |
| 8参数L-BFGS-B在多起点下不收敛 | 低 | 降至5参数（固定K_m₁=K_m₂=0, α=0）→纯MM+CSTR；8参数的L-BFGS-B在4000个递推步上几乎不可能失败 |
| 残差自相关→需要AR(2) | 中 | 正确：扩展NTU(t)为AR(2)→加β₃·NTU(t-2)项，参数+1。论文中作为"模型诊断改进"的叙事 |
