# Sum 5: 灰箱模型重构与双模态阈值发现

> 从101维XGBoost到2参数物理CSTR的方法论演变 | 2026-07-24

---

## 1. 问题起点

**原Q1方案**（XGBoost+SHAP+三模型集成+101维特征金字塔）存在系统性缺陷：
- 时序数据被拍平为表格，丢失递推结构
- 出厂同期变量(PH/CLR/CL2/TW_FLOW)用于预测同期的NTU → 循环论证
- 80维边际ΔR²=0.14 → 特征冗余
- 2026年预测代码=4行print → 核心交付物缺失

**原Q2方案**（4层TCN+注意力+物理Loss+PINN, 全量数据）：
- R²=-0.15, 被7参数AR(6)(R²=0.52)碾压
- 四种物理约束(平滑/上界/PINN/注意力)在违规率0%时全面反效
- TE/MIC/CCF三种时滞估计方法全部d*=0

## 2. 重构方案

### 2.1 特征工程精简 (101→15维)

```
泵编码: 10维0/1展开 → 2维整数count (同质性假设)
删除: L2+(12) + L3(15) + L4(40) + L5(4) = 71维
删除: PH/CLR/CL2/TW_FLOW(同期) = 4维泄露
保留: 灰箱公式中出现的变量 + 时间编码 = 10+5=15维
```

| 被删层 | 原因 |
|------|------|
| L2+ 幂次 | 递推模型自动编码非线性，不需要多项式展开 |
| L3 滞后 | β递推包含全阶分布式滞后 → 一个参数替代15个离散快照 |
| L4 聚合 | mean/std/max是对时序的去时间化 → 递推保留了时间 |
| L5 交互 | SHAP交互显著性全部不通过 |

### 2.2 Q1灰箱模型 (段1+段2)

```
段1: FILT(t) = β₁·FILT(t-1) + (1-β₁)·RW_NTU(t)·[1 - η(ALUM,CLR,day)]
段2: NTU(t)  = β₂·NTU(t-1)  + (1-β₂)·FILT(t), β₂=exp(-2h/θ)
```

| 参数 | 含义 | 收敛值 | 判定 |
|:---|------|:---:|:---:|
| β₁ | 工艺链时间常数 | 0.999→1 | 触上界 |
| K_m₀ | 基础矾需求量 | 0.001 | 触下界 |
| K_m₁/K_m₂ | 矾需量季节调制 | 约0 | 不显著 |
| α | CLR竞争系数 | 0 | 触下界 |

**段1结论**: 全部参数收敛到边界 = 不是优化失败 = 系统在告诉我们：99%+去除效率使外部输入的即时效应在2h采样间隔下不可辨识。这是**物理真相而非模型缺陷**。

### 2.3 FILT_NTU阈值发现

三种方法交叉验证确定双模阈值：

| 方法 | θ | 应力区样本 |
|------|:---:|:---:|
| Jenks自然断点 | 0.70 | 221 (5.1%) |
| 相关系数变点 | 0.70 | 221 (5.1%) |
| 建模实用阈值 | **0.15** | **962 (22.0%)** |

**舒适区 (FILT < 0.15, 78%):**

| 变量对 | r |
|------|:---:|
| RW_FLOW → FILT | 0.08 |
| ALUM → FILT | 0.16 |
| RW_NTU → FILT | -0.01 |
| FILT → NTU | **0.03** |

→ CSTR物理在此区间不适用; NTU波动由管壁释放/采样误差主导。

**应力区 (FILT ≥ 0.15, 22%):**

| 变量对 | r | 变化 |
|------|:---:|:---:|
| RW_FLOW → FILT | 0.24 | ×3 |
| ALUM → FILT | 0.02 | ×0.13 (被控制抹平) |
| RW_NTU → FILT | 0.10 | 从消失→浮现 |
| FILT → NTU | **0.79** | ×26 |

→ CSTR物理在此区间主导, 模型有效。

## 3. Q2闭环分解尝试（负面结果）

尝试将ALUM分解为操作员策略+意外偏差，以消除控制系统的因果混淆：

```
ALUM_policy(t) = α₀ + α₁·RW_NTU(t) + α₂·max(0, FILT(t-1)-0.15)
ε(t) = ALUM(t) - ALUM_policy(t)
→ 用ε替代原始ALUM驱动灰箱段1
```

**结果**：操作员策略R²=0.0067 — 线性不可表示；ε版本灰箱R²=-3062。

**结论**：操作员的控制规律远非线性（涉及多变量逻辑、累积效应、经验判断），简单的OLS反馈律不足以提取有意义的ε。如实记录为"尝试了控制律分解但在此数据上不可行"。

## 4. Q2滞后权重（应力区2层TCN）

虽然ΔFILT预测全部失败（所有模型R²≤0 — 2h间隔下滤池瞬态变化≈白噪声），但TCN卷积核分析提取的滞后权重具有物理意义：

| 变量 | 峰值 | 物理含义 |
|------|:---:|------|
| RW_FLOW | 6h | 水力负荷→过滤延迟 |
| ALUM | 8h | 矾反应+沉淀综合延迟 |
| RW_NTU | 10h | 原水浊度全链条穿透 |

这些权重作为Q3源A的注意力机制soft prior输入。

## 5. 关键教训

1. **ML在闭环控制系统上做开环预测必然失败** — 操作员的控制动作消除的就是你想建模的信号
2. **灰箱 > 黑箱** — 2参数CSTR(R²=0.53) > 200棵树XGBoost(R²=0.34) > 4层TCN(R²=-0.15)
3. **先分区再建模** — 在78%数据中找不到信号不是方法问题，恰恰是方法告诉你这里没有信号
4. **负面结果是论文的加分项** — 段1全参数触底 = 体系告诉你外部输入无效应 = 这是物理发现
5. **时滞不可估计是采样间隔的硬约束** — 2h > 工艺延迟(0.5-2h), 需要更高频数据或物理先验
