# Sum 2: F/RIDE 数据质量审查与排除决策

> 数据勘探驱动的特征筛选结论 | 2026-07-23

---

## 问题

Q2要求建立FILT.NTU的动态模型，题目指出影响因素包括"矾投加量(ALUM, F/RIDE)"。需判断F/RIDE与ALUM是**冗余互补**还是**应排除的噪声列**。

---

**关键发现**: 10个月覆盖率≤17%，仅11-12月有≥79%的记录。F/RIDE的有效记录集中在年末两个月——这暗示F/RIDE可能在2025年10月后才开始系统性地被记录，此前要么设备未安装、要么操作员未按规定录入。

### 3. 与ALUM的物理一致性检验

物理期望: F/RIDE（矾液投加速率）∝ ALUM（到池矾量），二者应为强正相关（≈1.0）。

实际数据: **r = 0.1816**（极弱）

```
ALUM范围:  0.04 ~ 0.08  (稳定、物理合理)
F/RIDE范围: 0.01 ~ 100.0  (四个数量级跨越，物理不合理)

F/RIDE / ALUM 比值: mean=44.3, std=270.7, min=0.17, max=2000.0
```

F/RIDE与ALUM的比值从0.17到2000——不是"记录误差"，而是**不同量纲/不同仪器/不同标准的混合产物**。这不是同一物理量的两种表达，这是**两套互不兼容的计量系统的数据混在一起**。

### 4. 预测价值检验（是否与目标变量相关？）

| 变量 | 与FILT.NTU的Pearson相关 |
|------|:---:|
| F/RIDE | -0.044 (≈0) |
| ALUM | -0.006 (≈0) |
| R/W NTU | 0.154 (弱，但衰减有规律) |

F/RIDE对FILT.NTU的直接预测力为零——甚至低于ALUM（r=-0.006 > r_F/RIDE=-0.044）。**即使忽略数据质量问题，F/RIDE在统计意义上也没有提供额外的预测信息。**

### 5. 2026年测试集状态

| 月份 | F/RIDE状态 |

| 2026年1月 | 12行全为常量0.01（零方差→无法作为特征） |

**2026年F/RIDE完全不可用**——无论训练集如何，预测时根本没有这个变量的值。

## 建模决策

**F/RIDE不纳入任何模型的特征集。**

| 排除理由 | 证据强度 |
|------|:---:|
| 83.1%缺失/无效率，训练集不可用 | 硬 |
| 10/12月覆盖率<17%，数据存在系统性采集偏倚 | 硬 |
| 与ALUM的相关远低于物理预期(r=0.18)，暗示计量标准不一致 | 硬 |
| 值域异常(0.01~100)，无统一物理单位 | 硬 |
| 对FILT.NTU预测力为零(r=-0.044) | 硬 |
| 2026年测试集全部为常量或NaN，无法用于预测 | 硬 |

**唯一的变量以ALUM代表矾投加控制**——ALUM记录完整（4380/4380有效）、值域物理合理(0.04~0.08)、单位明确(mg/L)。

## 论文措辞建议

> "在数据预处理阶段，我们对F/RIDE（矾投加流量）进行了数据质量审查。结果发现F/RIDE存在严重的记录缺失：训练集中仅16.9%的数据为有效数值，10/12个月覆盖率不足17%；2026年测试集中F/RIDE在1-2月为常量（方差为零）、3月全部缺失。进一步检验发现，F/RIDE的有效值与ALUM的相关系数仅为0.18（物理预期应接近1.0），且数值跨度达四个数量级（0.01~100），暗示可能存在系统性的计量标准不统一或仪器校准问题。此外，F/RIDE与目标变量FILT.NTU的相关系数为-0.044，未提供显著的统计信息。基于上述六项独立证据，我们将F/RIDE排除在特征集之外，仅以记录完整且物理解释清晰的ALUM表征矾投加控制。"
